## 核心模块

### 1. 进程模型
- **Master 进程** (`src/core/master.js`): 子进程管理、资源监控
- **Worker 进程** (`src/services/api-server.js`): HTTP 服务器、请求处理

### 2. 请求处理管道
```
HTTP Request
    │
    ▼
┌─────────────┐
│  CORS 处理  │
└─────────────┘
    │
    ▼
┌─────────────┐
│  静态文件   │
└─────────────┘
    │
    ▼
┌─────────────┐
│  插件路由   │
└─────────────┘
    │
    ▼
┌─────────────┐
│  认证流程   │
└─────────────┘
    │
    ▼
┌─────────────┐
│  API 分发   │
└─────────────┘
```

### 3. 提供商适配器架构
- **适配器模式**: 统一不同 AI 提供商的接口
- **策略模式**: 不同协议的转换策略
- **工厂模式**: 适配器实例创建

### 4. 提供商池管理
- **健康检查**: 定时检测提供商可用性
- **LRU 选择**: 最少最近使用策略
- **故障转移**: 自动切换到备用提供商
- **并发控制**: 限制每节点并发请求数

## 设计模式应用

| 模式 | 应用场景 | 文件 |
|------|----------|------|
| 适配器模式 | 提供商接口统一 | `src/providers/adapter.js` |
| 策略模式 | 协议转换 | `src/converters/strategies/` |
| 工厂模式 | 适配器创建 | `getServiceAdapter()` |
| 单例模式 | 管理器实例 | `PluginManager` |
| 观察者模式 | 事件广播 | `event-broadcast.js` |
| 责任链模式 | 认证流程 | 插件认证链 |

## 插件系统

### 插件生命周期
```
discoverPlugins()  →  发现插件
     │
     ▼
initAll(config)    →  初始化所有插件
     │
     ▼
middleware()       →  注册中间件
     │
     ▼
routes[]           →  注册路由
     │
     ▼
destroyAll()       →  销毁插件
```

### 插件类型
- **AUTH**: 认证插件，参与认证流程
- **MIDDLEWARE**: 普通中间件，不参与认证

## 配置体系

### 配置来源优先级
1. CLI 参数（最高优先级）
2. 配置文件 (`configs/config.json`)
3. 默认配置

### 关键配置项
| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `SERVER_PORT` | 服务器端口 | 3000 |
| `MODEL_PROVIDER` | 默认提供商 | gemini-cli-oauth |
| `MAX_ERROR_COUNT` | 不健康阈值 | 10 |
| `CRON_NEAR_MINUTES` | Token 刷新间隔 | 15 |
