## 🔧 修复的问题

### 1. 登录跳转问题
**问题**: 登录成功后使用 `router.push('/')` 但路由守卫检测到没有 token 又跳转回登录页

**修复**: 使用 `window.location.replace('/')` 确保 token 已保存到 localStorage 后再进行完整页面跳转

**文件**: `app-vue/src/views/core/Login.vue`
```javascript
// 修复前
router.push('/');

// 修复后
window.location.replace('/');
```

### 2. 401 响应拦截器问题
**问题**: `/api/system/check-update` 在未登录或 token 过期时返回 401，触发拦截器跳转到登录页

**修复**: 将可选端点加入白名单，其 401 错误不触发跳转

**文件**: `app-vue/src/utils/api.js`
```javascript
// 修复后
const optionalEndpoints = ['/api/system/check-update', '/api/validate-token'];
const isOptional = optionalEndpoints.some(endpoint => url.includes(endpoint));

if (!isOptional) {
  // 只有在非可选端点且不在登录页时才跳转
  if (currentPath !== '/login') {
    localStorage.removeItem('authToken');
    router.push('/login');
  }
}
```

### 3. 路由路径统一
**问题**: 多处使用 `/vue/login` 和 `/vue/` 路径，但 Vue Router 使用的是 `/login` 和 `/`

**修复文件**:
- `Login.vue`: `/vue/` → `/`
- `useAuth.js`: `/vue/login` → `/login`
- `api.js`: `/vue/login` → `/login`
- `NotFound.vue`: `/vue/` → `/`
- `Header.vue`: `/vue/login` → `/login`

### 4. API baseURL 配置
**问题**: 使用 `window.location.origin` 作为默认 baseURL，开发模式下请求发送到 5173 而不是通过代理

**修复**: `app-vue/src/utils/api.js`
```javascript
// 修复前
const createApiInstance = (baseURL = window.location.origin) => {

// 修复后
const createApiInstance = (baseURL = '') => {
```
